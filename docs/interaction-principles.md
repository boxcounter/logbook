# 交互原则（Interaction Principles）

本文件是 Logbook 前端交互的**权威约定**。新增弹层、编辑器、输入控件或任何涉及焦点/键盘/取消的交互时，先遵循这里的原则，不要逐组件另起一套。原则是"为什么"，下挂的「当前实现」是当下满足该原则的做法——实现可演进，原则不轻易改。

改动这些原则需在 PR 里说明理由。涉及视觉/布局的呈现，先出 mockup 再定（见各 owner 偏好）。

---

## 1. 不静默丢失用户输入

任何"离开 / 取消 / 切走"的路径都不得无声丢弃用户已输入但未保存的内容。

- 有未保存改动时：给一次明确的挽回机会（确认条），或自动保存——**绝不**静默丢弃。
- 无改动时：可以直接退出，不打扰。

**当前实现**
- `EntryRowEdit`：脏数据时 esc / 点击别处 / 失焦都弹就地确认条「放弃修改？[放弃][继续编辑]」；无改动直接退出。
- Day note：`blur` 即保存；esc 还原到进入编辑前的快照。

## 2. 消解一致且宽容（dismissal）

所有弹层与编辑器对「退出」的响应必须一致、可预期，并共用同一套机制——而不是每个组件各写各的。三条退出路径都要支持：**esc / 点击别处 / 失去焦点**。

- **点击别处**：document 级 `mousedown`（capture 阶段）监听，仅在打开时挂、关闭/卸载时移除；用一个包住「触发器 + 弹层」的 `ref` 判定，组件内部点击（含触发器，避免开→关→重开抖动）不自关。
- **esc**：最内层弹层用 window capture + `stopPropagation` 自己吃 esc（phase-aware：值→退回、维度→关闭）。编辑器用元素 `@keydown.esc.stop` 处理"焦点在内"的情况，外加一个 document `keydown` 兜底，仅当 `activeElement` 在编辑器之外时触发——否则焦点在别处时 esc 会丢失。
  - **铁律（已踩坑三次）**：元素级 `@keydown` 只在焦点落在组件内部时才收得到 esc。任何弹层/对话框/编辑器，要么**开启时把焦点移进自己**（如 modal / QuickJumpPopover：`onMounted`/`watch(open)` 里 `el.focus()`，根元素需 `tabindex="-1"`），要么挂 **window/document 级监听**（如 DimensionPopover）。绝不能只靠元素 `@keydown` 还假设焦点恰好在里面——否则从触发按钮或别处按 esc 会无效。
- **失去焦点**：document `focusin`，目标落在编辑器之外即消解（覆盖切窗回来焦点被主输入框抢走等场景）。

新组件应复用以上模式，而非发明新机制。

## 3. 键盘优先，快捷键按使用频率分配

可用键盘完成的高频操作必须有快捷键；**越常用的操作，按键越简单**（修饰键越少），不常用的才叠加 Shift 等修饰键。

**当前实现**
- `⌘[` / `⌘]` = 上一天 / 下一天（最高频）
- `⌘⇧[` / `⌘⇧]` = 上个月 / 下个月（次高频）
- `⌘T` = 回到今日并聚焦输入框
- 提示栏只展示与当前控件相关的快捷键，全局导航快捷键不挤进输入框提示。

## 4. 尊重输入上下文

帮用户省一步，但不要打断用户。

- 窗口重新聚焦时自动聚焦主输入框——**除非**用户正在编辑某个字段（input / textarea / select / contenteditable），此时不抢焦点。
- `Enter` 提交类处理必须对 IME 安全：用 `e.isComposing` 守卫，中文等输入法回车选词不可被当作提交。
- 自动聚焦/自动跳转后，把光标放到用户接下来最可能操作的位置。

---

## 适用范围

- 这些原则覆盖前端所有交互组件；后端约定见 `src-tauri/CLAUDE.md`，技术规格见 `SPEC.md`。
- 评审新功能时（requesting-code-review / 手测），按本文件逐条核对，而不是只看单个组件是否"能用"。
