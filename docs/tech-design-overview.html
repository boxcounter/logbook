Tauri 2.x Rust Vue 3 TypeScript

架构分层

UI 用户界面，跑在系统 WebView 中 (macOS WKWebView) Vue 3 · TypeScript · Tailwind CSS
invoke('command', params) ────→ ←──── JSON (自动序列化)
胶水层 IPC 通道 · 窗口管理 · 文件监听 → 事件推送 · 系统对话框 Tauri 2.x · plugin-dialog · plugin-opener
函数调用 ────→ ←──── Result
后端 文件 I/O · YAML 解析 · Duration 计算 · 统计聚合 · Config 校验 Rust · serde_yml · notify · chrono · regex · uuid

类比：Tauri = 用 Web 技术写 UI 的 Electron，但后端用 Rust 而非 Node.js，体积小、性能好、不连网、纯本地。

Tauri 自动完成 camelCase ↔ snake_case 转换和 JSON 序列化。参数名、类型即契约，无需独立 API schema 文件。

日文件格式：

---
note: 五一假期

  - id: a1b2c3d4-...
    item: 准备技术方案

核心数据流

启动流程

App.vue mounted → invoke('init') → 读 root_path.txt → InitResult

NeedsSetup → SetupScreen （文件夹选择器） → set_root_path → 重新 init

ConfigError(errors) → ConfigErrorBanner 显示错误列表

Ready { config, today, commitments } → 渲染 TodayView

录入流程

用户输入文字 → 前端 regex 扫描 duration 求和 → 去除匹配片段 → item → 追加继承维度 → invoke('append_entry', ...) → Rust parse_duration → u32 · 原子写入 → 返回 Entry · 刷新列表 + Commitments

文件监听

Tauri setup hook 启动 notify 线程 → 递归 watch root → config.yaml / _monthly.md 变更 → emit config-changed / commitments-changed 事件 → 前端自动刷新

关键设计决策

  • 文件名即日期真源 frontmatter 不存 date，避免二义性。文件路径 {root}/2026/06/2026-06-07.md 即唯一真相。
  • 乐观删除 + Toast 撤销 点 × → entry 立即从列表消失 → 5 秒 toast "Undo"。超时才调 Rust delete。避免确认弹窗打断流程。
  • 双击统一编辑 Item 文字和 Duration 都是双击进入编辑。Duration 支持 +30 / -15 增量语法，Enter 确认，Esc 取消。
  • Dimension 继承（纯前端） 上一条的维度自动带入下一条。lastDimensions ref 记忆，不存磁盘。UI 行为而非存储关注点。
  • Entry.id = UUID v4 不用数组 index 作 key。删除时避免 React/Vue 的 key 错位问题。前端 list 用 entry.id 渲染。
  • UpdateEntry 全部可选 部分更新（item? / duration? / dimensions?），避免全量 NewEntry 导致覆盖陈旧数据的风险。
  • 最多 1 个 source: monthly 维度 当前只有 Goal。值列表不从 config 取，从 _monthly.md 的 goals 并集动态构建。
  • 原子写入 所有文件操作先写 temp 再 rename。防止并发读写导致的文件损坏。
